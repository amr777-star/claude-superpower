**Framework Selection:**
- Pre-seed/Seed: YC format (10 slides, traction-first, brutal simplicity)
- Series A: Sequoia format (12 slides, narrative-driven)
- Series B+: Hoffman format (15 slides, data + vision + execution proof)
- Quick pitch: Kawasaki 10/20/30 (10 slides, 20 min, 30pt font)

**SEQUOIA FORMAT (12 Slides) — Default for Series A**

**Slide 1 — Company Purpose**
- One sentence: "[Company] is the [category] that [unique value] for [customer] so they can [outcome]"
- Logo, tagline if compelling. No clutter.

**Slide 2 — Problem**
- Who has the problem? How severe? How do they solve it today (badly)?
- Quantify the cost of the problem: "$X lost per year" or "Y hours wasted"
- Use a customer quote or real scenario to make it visceral
- Rule: Make the investor FEEL the pain before they see the cure

**Slide 3 — Solution**
- Your value proposition in plain language
- 2-3 concrete use cases showing the product in the customer's workflow
- Product screenshot or visual if available
- This is the "aha moment" slide

**Slide 4 — Why Now**
- Recent change that makes this inevitable: technology inflection, regulatory shift, behavioral change, cost threshold crossed
- Formula: "[Trend X] has been building for [Y] years. [Development Z] now makes our solution possible/necessary."
- If you cannot answer compellingly, OMIT this slide (absence > weak answer)

**Slide 5 — Market Size**
- TAM (top-down, cite source): Total if 100% market share
- SAM (filtered): Portion reachable with current model
- SOM (bottom-up): Realistic 3-5 year capture
- Show BOTH top-down calculation AND bottom-up (customers x ARPU) math
- Never inflate TAM — investors see through it immediately

**Slide 6 — Competition**
- 2x2 positioning matrix with axes showing YOUR key differentiating dimensions
- OR feature comparison table (you vs top 3-4 competitors)
- NEVER dismiss competitors. Show what they do well, then why you win specifically.
- "We are the only [category] that [differentiator] for [customer], whereas competitors [limitation]"

**Slide 7 — Product**
- Key features mapped to problems (not a feature laundry list)
- Screenshots, architecture highlights, IP
- Development roadmap: what exists today vs. what's being built
- Focus on the 2-3 capabilities that deliver core value

**Slide 8 — Business Model**
- Revenue mechanism: subscription, usage-based, transaction fee, marketplace take rate
- Unit economics: CAC, LTV, LTV:CAC, payback period, gross margin
- Pricing tiers if applicable

**Slide 9 — Traction**
- The most important slide if you have ANY metrics
- Revenue/MRR chart (hockey stick if possible)
- Customer logos, NPS, retention rates
- Growth rate (MoM or QoQ) — "growth is the only thing that matters"
- If pre-revenue: LOIs, waitlist, pilots, qualified pipeline

**Slide 10 — Team**
- Founder-market fit: WHY are YOU the right team?
- Relevant prior experience ONLY (not a resume dump)
- Domain expertise, prior exits, technical credentials
- Key advisors if genuinely active and credible

**Slide 11 — Financials**
- 3-year P&L summary (Revenue, Gross Profit, OpEx, EBITDA, Net Income)
- Key assumptions in footnotes
- Path to profitability or key inflection points

**Slide 12 — The Ask**
- Specific dollar amount
- Use of funds: "Engineering X%, Sales Y%, Marketing Z%"
- Milestones this funding enables (18-24 month horizon)
- What you will have achieved by the time you raise again

**YC FORMAT (10 Slides) — For Pre-Seed/Seed**
Same structure but compressed: Cover, Problem, Solution, Why Now, Market, Product, Traction (SLIDE 7 — front-loaded), Business Model, Team, The Ask

**KAWASAKI 10/20/30:**
Title, Problem/Opportunity, Value Proposition, Underlying Magic, Business Model, Go-to-Market, Competition, Team, Financials+Metrics, Status+Ask

### STORYTELLING FRAMEWORKS

Apply the best framework for the audience:

**AIDA (Attention-Interest-Desire-Action):**
- Slides 1-2: Hook with surprising stat or bold claim
- Slides 3-5: Problem context that resonates
- Slides 6-8: Solution + proof (testimonials, metrics)
- Slides 9-10: Clear CTA with timeline

**Minto Pyramid (for board decks/exec presentations):**
- Level 1 (top): Single governing recommendation
- Level 2: 3-5 MECE supporting arguments
- Level 3: Evidence and data for each argument
- Present top-down: Answer first, then support

**SCQA (McKinsey opening):**
- Situation: Stable facts everyone agrees on
- Complication: What disrupts the situation
- Question: The natural question this provokes
- Answer: Your recommendation (immediately, not at the end)

**PAS (Problem-Agitation-Solution):**
- Problem: Name the specific pain
- Agitation: Deepen consequences — "what happens if unsolved?"
- Solution: Present as relief, not as a product pitch

**Hero's Journey (customer as hero, you as guide):**
Ordinary World → Call to Adventure → Resistance → Meeting the Guide (you) → Trials (proof) → Transformation (results) → Return (broader impact)

### SALES DECKS

**Discovery Deck (Meeting 1, 6 slides):**
Company Overview → Agenda → Customer Story → Common Challenges → Diagnostic Questions → Next Steps
Rule: Listen 70%, present 30%

**Demo Deck (Meeting 2-3, 11 slides):**
Agenda → What We Heard (recap) → Core Problem (their words) → Our Approach → Product Demo (3-4 flows using Tell-Show-Tell) → Customer Results → Implementation Overview → Next Steps

**Proposal Deck (Meeting 3-4, 10 slides):**
Exec Summary (SCQA) → Their Goals (verbatim) → Current Gaps → Proposed Solution → Implementation Plan → ROI Analysis → Customer Evidence → Pricing Options → Risk Mitigation → Decision Criteria + Timeline

**ROI Calculator Slide:**
```
Current Cost = (Manual hours x rate) + (Error rate x cost) + (Churn attributable) = $TOTAL
Investment = License + Implementation = $INVEST
Net Year 1 = $TOTAL - $INVEST
3-Year ROI = (Net1 + Net2 + Net3) / $INVEST x 100%
```

### BOARD DECKS

**Structure (15-20 slides):**
Section A — Status: CEO State of Company → KPI Scorecard → Highlights → Lowlights → OKR Progress
Section B — Financials: P&L vs Plan → Cash & Runway → Revenue Breakdown → Pipeline
Section C — Functional: Product Roadmap → Hiring → GTM Update

## Slide Design Rules
- **One idea per slide** — if you can't state it in one sentence, split it
- **Action titles** — "Market is growing 40% YoY driven by AI adoption" NOT "Market Overview"
- **McKinsey anatomy**: Action Title (top) → Body (evidence) → Source (footer)
- **30pt minimum** font for all text
- **Charts over tables** — show trends, not numbers
- **Max 4 colors** in any chart
- **Direct label data** — eliminate separate legends
- **Ghost deck first** — write all action titles before any design. Read titles alone = the full story.
